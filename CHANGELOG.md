## [3.2.0] — 2026-04-11

### Добавлено
- **CLI аргументы**: `--load`, `--resume`, `--difficulty`, `--depth`, `--black`, `--version`
- **Визуальная подсказка**: плавная зелёная пульсация рамки вокруг шашки, обязанной бить
- **AI: транспозиционная таблица** (Zobrist hashing, 500K записей) — кеширование позиций
- **AI: iterative deepening** — поиск с глубины 1 до N с лучшим ходом первым
- **AI: quiescence search** (глубина 6) — разрешение взятий за горизонтом
- **AI: Late Move Reduction** — ускорение эндшпиля с дамками
- **AI: killer move heuristic** — ходы, вызвавшие отсечение, пробуются первыми
- **AI: детекция повторов** — позиции в дереве поиска оцениваются как ничья
- **AI: эндшпильные паттерны** — мгновенное распознавание ничьи (дамка vs дамка)
- **AI: расстояние дамки до противника** — целенаправленные ходы вместо хаотичных
- **Оценка**: связанные пешки, фазовые веса (дебют/эндшпиль), золотые углы, централизация дамок
- **Color enum** (`Color.BLACK`, `Color.WHITE`) — типобезопасная замена строк `"b"`/`"w"`
- **AIEngine класс** — инкапсуляция параметров AI поиска
- Правила версионирования и обновления документации в CLAUDE.md
- 11 тестов CLI, 2 теста превращения при взятии, 6 тестов AIEngine

### Исправлено
- **Превращение при взятии**: пешка теперь продолжает бить как дамка (правила русских шашек)
- **Отображение превращённой фигуры**: проверка дамочного ряда на любой точке пути, а не только финальной
- Дедупликация `dangerous_position()` — одна реализация вместо двух

### Изменено
- Цветовая схема: зелёный = выбранная шашка, магента = промежуточные позиции
- Vectorized evaluation: `bincount` + `dot product` вместо `count_nonzero` + `np.where`
- `_evaluate_fast` с pre-flattened lookup таблицами (без аллокации на каждый вызов)

### Производительность
- Opening depth 7: 116ms → **16ms** (7.3x)
- Endgame 2v2 kings depth 7: 23.8s → **2.6s** (9.1x)
- Endgame 3v3 depth 7: 252ms → **83ms** (3x)

## [3.0.0] — 2026-04-10

### Добавлено
- README.md с полной документацией проекта
- Единый source of truth для версии: `draughts/__init__.py`

### Изменено
- Минимальный UI: только доска + стандартное меню Windows
- Удалены: splash screen, анимации, звуки, status bar, таймер, база обучения
- Чистый алгоритмический AI (alpha-beta minimax) без Monte Carlo

### Удалено
- Learning database (AI теперь чисто алгоритмический)
- Звуковые эффекты
- Splash screen и анимации загрузки
- Панель захваченных фигур и status bar
- Настройка скорости (таймер был отключён)

## [2.0.0] — 2026-04-09

### Добавлено
- **NumPy движок**: доска как `np.ndarray((8,8), dtype=np.int8)` со знаковой кодировкой
- **Alpha-beta minimax** с настраиваемой глубиной поиска (1-10)
- Адаптивная глубина в зависимости от фазы игры
- Штраф за подставки (ходы, после которых противник может бить)
- Move ordering: захваты первыми для лучшего отсечения
- Статическая оценка: материал + продвижение + центр + мобильность + угрозы

### Изменено
- **Premium UI**: процедурные текстуры дерева, 3D-шашки с освещением и тенями
- Доска с деревянной рамкой и координатными метками
- Переработан layout: бесшовная компоновка с felt-панелью

### Исправлено
- AI больше не отдаёт фигуры бесплатно (корректные веса материала + safety penalty)
- Исправлен crash AI при оценке дамок
- Исправлена очистка потока AI (segfault)

## [1.0.0] — 2026-04-08

### Добавлено
- Полный порт с Borland Pascal 7.0 на Python 3.12 / PyQt6
- Правила русских шашек: обязательное взятие, множественные прыжки, дамки
- Игровой контроллер: управление ходами, AI-поток, save/load
- UI: доска, главное окно, диалоги (настройки, справка, об авторе)
- Просмотр партии с элементами управления
- Сохранение/загрузка партий в JSON
- Monte Carlo AI с базой обучения
- 54 теста для правил доски
- Линтер (ruff) и типизация (mypy)
