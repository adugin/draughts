## [Unreleased]

### Добавлено (M6 — Data generation & distribution)
- **Скачивание расширенной базы эндшпилей** (меню «Настройки → Скачать расширенную базу эндшпилей»): прямая загрузка 4-piece bitbase (~126 МБ сжатого) с GitHub Releases. SHA-256 проверка, прогресс-бар, отмена, атомарная запись. Файл сохраняется в `%APPDATA%/DRAUGHTS/bitbase/` и подхватывается движком при следующем запуске без переустановки.
- **Меню «Инструменты»** (D36) — инструменты для генерации контента прямо в программе:
  - **Импорт книги из PDN** — построение/расширение дебютной книги из пользовательских PDN-файлов (multi-select, настройка полуходов 4–40). Веса накапливаются при повторных позициях.
  - **Добыча тактических задач** — сыграть N self-play партий (уровень 1 vs 6, рандомные дебютные ходы) и извлечь ошибки как тренировочные задачи. Настраиваемое число партий (5–500) и seed для воспроизводимости.
  - Прогресс немодальный: игра продолжается, можно отменять в любой момент, частичный результат корректно сохраняется.
- **Спецификация bitbase-формата** (`docs/BITBASE_FORMAT.md`): полное описание контейнера, Zobrist-хеша, правил игры для которых действительна база, механизма целостности, размеров и планов развития до v2.0.

### Добавлено (M5 — Ecosystem & Depth)
- **Смена стороны в партии** (Ctrl+F) — поменяться цветом с компьютером на своём ходу. Доска переворачивается (шашки игрока снизу), позиция и история PDN остаются абсолютными, координаты a1..h8 привязаны к полям. AI отвечает один раз за свой новый цвет; повторный Ctrl+F блокируется до завершения его хода.
- **Сдача партии** — меню «Игра → Сдаться». Корректно инвалидирует фоновый ход AI, эмитит game_over и очищает выбор.
- **Модель дерева партии** (backend M5.a) — `draughts.game.gametree.GameNode`/`GameTree` поддерживают основную линию + варианты, комментарии и NAG (`!`, `??`, `!?`, `$N`). PDN-парсер и writer работают с RAV: варианты в скобках `(...)`, комментарии `{...}` сохраняются при round-trip.
- **Панель дерева вариантов** (F4) — дополнительная док-панель: показывает структуру загруженного PDN-дерева, подсвечивает текущий ход, клик по узлу основной линии переходит к этой позиции.
- **База партий PDN** (Ctrl+D) — браузер многопартийных PDN-файлов: сортируемая таблица (White/Black/Result/Event/Date/Round/Moves), фильтр по тексту, двойной клик загружает партию.
- **Подсказка с principal variation** (Ctrl+H) — показывает до 4 ходов AI-продолжения. Глубина поиска увеличена с 4 до 6.
- **FEN clipboard** (Ctrl+Shift+C / Ctrl+Shift+V) — скопировать позицию в буфер обмена / вставить из буфера.
- **DXP-протокол** — `python -m draughts.engine --dxp --port 27531` запускает сервер FMJD Draughts eXchange Protocol. Поддерживает GAMEREQ, GAMEACC, MOVE, GAMEEND. Совместимость с внешними GUI (CheckerBoard, Dam 3.0) и турнирными платформами.
- **Авто-конверсия JSON → PDN** — при запуске сканируется папка данных; старые JSON-сохранения получают PDN-копии для чтения внешними инструментами.
- **Импорт книги из PDN** — `python -m draughts.tools.import_book_from_pdn`: собирает первые N ходов из множества партий в OpeningBook. Веса накапливаются при повторных позициях.
- **i18n-инфраструктура** — модуль `draughts.i18n` с `_()` обёрткой, каталогами `draughts/locale/{ru,en}/`, компилятором `.po → .mo` (`python -m draughts.tools.compile_translations`). Переключение локали через `DRAUGHTS_LOCALE=en`.
- **Дебютная книга расширена до ~47 000 позиций** (было 1572) — покрытие всех основных дебютных линий; 3.7 MB (в рамках ≤ 5 MB бюджета).
- **Каталог задач расширен до 129** (было 30) — автомайнинг из асимметричных self-play партий (уровень 1 vs 6), депо PDN с сохранением в `draughts/resources/russian_draughts_puzzles.json`.
- **Конфигурируемый размер хеш-таблицы** — `hash_size_mb` из настроек применяется к движку (минимум 50K записей, масштабируется по байтам на запись).

### Изменено
- **Редактор позиции** — хоткей `E` → `Ctrl+E` (Windows-конвенция: функциональные команды требуют модификатор).
- **Меню «Анализ»** дополнено пунктом «Дерево вариантов» (F4).
- **Меню «Игра»** дополнено пунктами «База партий (PDN)» (Ctrl+D), «Сменить сторону» (Ctrl+F), «Сдаться».

### Исправлено
- `undo_move` очищает загруженное дерево вариантов — предотвращает сохранение несогласованного RAV после отмены.
- `load_game_from_pdn` сбрасывает дерево при невозможности воспроизвести ход, чтобы re-export не выдавал битые варианты.
- `compute_pv` защищён от отрицательных depth/pv_length.
- `_on_ai_finished` находит и очищает устаревшие Qt-потоки через `_pending_ai` (по generation), а не через `self.sender()` — предотвращает утечку потоков при гонке.
- Весь код прошёл через строгий ruff + mypy (strict на ядре + engine) + bandit HIGH; багов не найдено.

## [4.0.0] — 2026-04-13

### Добавлено
- **Модульная архитектура** — `ai.py` (1141 строка) разделён на 6 модулей: `ai/search.py`, `ai/eval.py`, `ai/tt.py`, `ai/moves.py`, `ai/state.py`, `ai/elo.py`
- **Движковый протокол** — `python -m draughts.engine` (UCI-подобный): position, go, stop, setoption, info, bestmove
- **PDN 3.0** — полная поддержка чтения/записи партий в формате Portable Draughts Notation с `[GameType "25"]` для русских шашек
- **FEN** — парсинг и генерация FEN-строк для русских шашек 8×8; копирование/вставка через меню Позиция (Ctrl+Shift+C/V)
- **Дебютная книга** — 1572 позиции из self-play, O(1) lookup по Zobrist-хешу
- **Эндшпильная база** — 399280 позиций (3 фигуры), WLD retrograde analysis
- **Texel-тюнинг** — автоматическая оптимизация весов оценки через scipy
- **Правила ничьей FMJD** — 1K vs 1K, 2K vs 1K (треугольник Петрова), 15-ходовое правило для дамок, 30-ходовое правило без прогресса, 3-кратное повторение позиции
- **6 уровней сложности** с Elo-рейтингами (~800–~2200+) вместо Любитель/Нормал/Профессионал
- **Неограниченная отмена** хода на всех уровнях сложности
- **Подсказка хода** (Ctrl+H) — подсветка лучшего хода с оценкой
- **Панель анализа** (F3) — оценка, лучший ход, глубина, время в реальном времени
- **Анализ партии** — автоматические аннотации (!!, !, ?!, ?, ??) с графиком оценки
- **Тренажёр задач** (Ctrl+P) — 30 тактических задач с автомайнингом из проанализированных партий
- **Редактор позиций** (E) — расстановка фигур, импорт/экспорт FEN, валидация, авто-превращение пешек
- **7 TOML-тем** — dark_wood, classic_light, nord_frost, dracula, gruvbox_dark, catppuccin_mocha, solarized_dark
- **Координаты** — a-h / 1-8 с Segoe UI, тенями и оптическим выравниванием
- **Hover-подсказки** — точки возможных ходов при наведении курсора (с учётом обязательного взятия)
- **Подсветка последнего хода** — рамка на клетках from/to
- **Индикатор раздумий** — "Думаю..." в заголовке окна + курсор ожидания
- **Сохранение настроек** — preferences в settings.json, загружаются при старте
- **Централизованное управление меню** — матрица состояний для всех action'ов

### Исправлено
- **Краш при анализе** — QThread deleteLater без quit/wait вызывал segfault
- **AI подставлялся** в эндшпиле 2K vs 1K (flat contempt вместо позиционной оценки)
- **Застревание при правом клике** — нестандартная механика промежуточных позиций заменена на left-click-only
- **Сохранение крашилось** на difficulty 4-6 (валидация 1-3 не обновлена после расширения)
- **Game-over проверял не ту сторону** после хода AI
- **Анализатор партий** — сравнение нотаций разных форматов, завышенный порог аннотаций
- **Drawn endgame** — 2K vs 1K ошибочно считался выигрышем
- **TT-флаги** — min-узлы всегда получали LOWER вместо EXACT (rate 11% → 29%)
- **Hover-точки** показывались для шашек, которые не могут ходить (обязательное взятие)
- **Disabled пункты меню** визуально не отличались от enabled (добавлена тематическая стилизация)
- **Настройки терялись** — `use_tuned_eval` сбрасывался при каждом открытии опций
- **Счётчики ничьей** не сбрасывались при загрузке партий

### Изменено
- **Ввод ходов** — только левый клик (как lichess/CheckerBoard); правый клик = отмена выбора
- **Дефолтные настройки** по стандартам индустрии: difficulty 3 (~1400), подсветка хода ON, hover ON, координаты ON
- **Удалён таймер** — кумулятивное время не актуально для человек-vs-AI
- **About dialog** — динамические версия и автор из `__version__` / `__author__`
- **APP_NAME** и `__author__` — вынесены в константы, убран хардкод

### Удалено
- **Правый клик для промежуточных позиций** — нестандартная механика, вызывала баг
- **Таймер (D19)** — тулбар, сигналы, счётчики, настройка show_clock
- **messages.txt** — случайные фразы AI при раздумьях

### Архитектура

├── app/controller.py    # Qt glue: game state, turns, AI thread
├── game/
│   ├── board.py         # Board + rules + check_game_over (shared)

## [3.2.0] — 2026-04-11

### Добавлено
- **CLI аргументы**: `--load`, `--resume`, `--difficulty`, `--depth`, `--black`, `--version`
- **Визуальная подсказка**: плавная зелёная пульсация рамки вокруг шашки, обязанной бить
- **AI: транспозиционная таблица** (Zobrist hashing, 500K записей) — кеширование позиций
- **AI: iterative deepening** — поиск с глубины 1 до N с лучшим ходом первым
- **AI: quiescence search** (глубина 6) — разрешение взятий за горизонтом
- **AI: Late Move Reduction** — ускорение эндшпиля с дамками
- **AI: killer move heuristic** — ходы, вызвавшие отсечение, пробуются первыми
- **AI: детекция повторов** — позиции в дереве поиска оцениваются как ничья
- **AI: эндшпильные паттерны** — мгновенное распознавание ничьи (дамка vs дамка)
- **AI: расстояние дамки до противника** — целенаправленные ходы вместо хаотичных
- **Оценка**: связанные пешки, фазовые веса (дебют/эндшпиль), золотые углы, централизация дамок
- **Color enum** (`Color.BLACK`, `Color.WHITE`) — типобезопасная замена строк `"b"`/`"w"`
- **AIEngine класс** — инкапсуляция параметров AI поиска
- 11 тестов CLI, 2 теста превращения при взятии, 6 тестов AIEngine

### Исправлено
- **Превращение при взятии**: пешка теперь продолжает бить как дамка (правила русских шашек)
- **Отображение превращённой фигуры**: проверка дамочного ряда на любой точке пути, а не только финальной
- Дедупликация `dangerous_position()` — одна реализация вместо двух

### Изменено
- Цветовая схема: зелёный = выбранная шашка, магента = промежуточные позиции
- Vectorized evaluation: `bincount` + `dot product` вместо `count_nonzero` + `np.where`
- `_evaluate_fast` с pre-flattened lookup таблицами (без аллокации на каждый вызов)

### Производительность
- Opening depth 7: 116ms → **16ms** (7.3x)
- Endgame 2v2 kings depth 7: 23.8s → **2.6s** (9.1x)
- Endgame 3v3 depth 7: 252ms → **83ms** (3x)

## [2.0.0] — 2026-04-09

### Добавлено
- **NumPy движок**: доска как `np.ndarray((8,8), dtype=np.int8)` со знаковой кодировкой
- **Alpha-beta minimax** с настраиваемой глубиной поиска (1-10)
- Адаптивная глубина в зависимости от фазы игры
- Штраф за подставки (ходы, после которых противник может бить)
- Move ordering: захваты первыми для лучшего отсечения
- Статическая оценка: материал + продвижение + центр + мобильность + угрозы

### Изменено
- **Premium UI**: процедурные текстуры дерева, 3D-шашки с освещением и тенями
- Доска с деревянной рамкой и координатными метками
- Переработан layout: бесшовная компоновка с felt-панелью

### Исправлено
- AI больше не отдаёт фигуры бесплатно (корректные веса материала + safety penalty)
- Исправлен crash AI при оценке дамок
- Исправлена очистка потока AI (segfault)

## [1.0.0] — 2026-04-08

### Добавлено
- Полный порт с Borland Pascal 7.0 на Python 3.12 / PyQt6
- Правила русских шашек: обязательное взятие, множественные прыжки, дамки
- Игровой контроллер: управление ходами, AI-поток, save/load
- UI: доска, главное окно, диалоги (настройки, справка, об авторе)
- Просмотр партии с элементами управления
- Сохранение/загрузка партий в JSON
- Monte Carlo AI с базой обучения
- 54 теста для правил доски
- Линтер (ruff) и типизация (mypy)
