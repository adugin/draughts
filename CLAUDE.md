# DRAUGHTS — контекст проекта для Claude

## Цель проекта

Переписать игру **русские шашки** с **Borland Pascal 7.0** (~1998–2000) на **Python**.  
Исходники — DOS-программа с VGA-графикой, мышью, ИИ-противником и системой сохранений.  
Полный технический разбор оригинала: **[ANALYSIS.md](ANALYSIS.md)**  
Экранная раскладка (640x480, все координаты): **[SCREEN_LAYOUT.md](SCREEN_LAYOUT.md)**

## Ключевые факты об оригинальном коде

### Доска
- `array[1..8, 1..8] of char`, индексация `field[y, x]`
- Используются только тёмные клетки: `x mod 2 ≠ y mod 2` → 32 активные клетки
- Символы: `'b'`=чёрная шашка, `'B'`=чёрная дамка, `'w'`=белая шашка, `'W'`=белая дамка, `'n'`=пусто
- Начальная позиция (32 символа): `bbbbbbbbbbbbnnnnnnnnwwwwwwwwwwww`
- `y=1` — верх (чёрные), `y=8` — низ (белые)
- Нотация: столбцы `a–h`, строки `8–1` сверху вниз → `(x=1,y=1)=a8`, `(x=8,y=8)=h1`

### Правила
- Русские шашки: шашки ходят только вперёд, дамки — на любое расстояние
- Обязательное взятие, множественные прыжки, взятая фигура убирается после серии
- Превращение в дамку при достижении последнего ряда

### ИИ (компьютер играет чёрными)
Три уровня приоритета, вызываются последовательно:
1. **`SeeBeat()`** — обязательные взятия: Monte Carlo (1000 итераций), выбирает серию с максимальным счётом
2. **`Combination()`** — тактические жертвы: ищет ход, после которого белые вынуждены взять, и проверяет, выгоден ли ответный удар
3. **`Action()`** — обычные ходы: перебор с эвристической оценкой (для дамок тоже Monte Carlo)

Самообучение: сохраняет хорошие/плохие позиции в `goodbase.dat` / `badbase.dat` (текстовые файлы, по 32 символа на строку).

### Формат сохранений (`.sav`)
```
1              ← difficulty (1/2/3)
1              ← speed (1/2/3)
TRUE           ← remind (подсказка взятия)
1              ← language (1=рус, 2=eng)
FALSE          ← soundeffect
0.750          ← pause (коэффициент задержки)
<32 символа>   ← позиция после хода 0 (начало игры)
<32 символа>   ← позиция после хода 1
...
               ← пустая строка
<32 символа>   ← movie (те же позиции для просмотра)
...
```

## Принятые решения по переписыванию

- **Язык / фреймворк**: Python 3.12+ / PyQt6 (единственная зависимость)
- **UI**: современный (стандартные виджеты), компоновка близка к оригиналу, без попиксельного DOS-стиля
- **Окно**: изменяемый размер с пропорциональным масштабированием
- **Язык интерфейса**: только русский
- **Звук**: современные приятные эффекты, процедурно сгенерированные WAV, через QMediaPlayer
- **Анимации**: полный набор (заставка с молнией, перемещение шашек, TV-эффект и т.д.)
- **Автор оригинала**: упоминание в заставке и «О программе» — «Based on original by Dugin Andrew (1998–2000)»
- **Фразы ИИ**: оригинальные 38 фраз из MESSAGE.TXT
- **Форматы**: JSON (без совместимости со старыми .sav)
- **Все функции оригинала**: обучение ИИ (4 чекбокса), таймер, конфискация, инверсия цвета, отмена хода (только «Любитель»), просмотр партии
- `IZVRAT.PAS` (10×10) — реализовывать только после завершения 8×8 и по отдельному запросу

## Текущее состояние (2026-04-10)

**Игра работает и играбельна.** Все основные функции реализованы и протестированы.

### Исправленные баги (в этой сессии):
- QThread segfault при уничтожении AI-потока → `quit()` + `wait()` + `deleteLater()`
- AI зависал в Monte Carlo (бесконечный цикл) → добавлен `max_attempts=100`
- `IndexError` в `_exist()` при оценке ходов дамки → проверка `_between()` в цикле
- Серийное взятие правой кнопкой не работало → виртуальная доска для промежуточных прыжков
- AI crash не возвращал управление UI → `try/except` в `AIWorker.run()`

### Что ещё можно доработать:
- Заставка (splash): выравнивание шрифтов, визуальная полировка
- Анимации: пока не подключены к реальным ходам (шашки перемещаются мгновенно)
- Звуки: сгенерированы, но `sound_effect=False` по умолчанию — не тестировались в игре
- Конфискация шашки при нарушении обязательного взятия — логика не полностью реализована (сейчас просто запрещает ход)
- Масштабирование: нижняя панель ограничена 20% высоты окна — при очень больших окнах шашки внизу мельче, чем на доске
- `run_debug.py` содержит дебаг-логирование; в контроллере остались debug-логи (можно убрать позже)
