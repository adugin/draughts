# DRAUGHTS — контекст проекта для Claude

## Цель проекта

Русские шашки на **Python 3.12+ / PyQt6 / NumPy**.
Оригинал: Borland Pascal 7.0 (~1998–2000), полный разбор: **[ANALYSIS.md](ANALYSIS.md)**

---

## Команды разработки

```bash
# Запуск игры
python main.py                          # новая игра
python main.py game.json                # загрузить партию
python main.py --resume                 # продолжить из автосохранения
python main.py --difficulty 3 --black   # сложность + играть чёрными
python main.py --depth 8               # ручная глубина AI

# Бенчмарк AI (глубина 1-8, профилирование)
python benchmark.py --depth 8 --profile

# Тесты (запускать перед коммитом)
python -m pytest tests/ -q

# Линтер + форматтер (ruff)
ruff check draughts/ tests/
ruff check --fix draughts/ tests/
ruff format draughts/ tests/
```

Конфигурация ruff и mypy — в `pyproject.toml`.

## Архитектура

- **Доска**: `np.ndarray((8,8), dtype=np.int8)`, 0-индексация, знаковая кодировка: `BLACK=1, BLACK_KING=2, WHITE=-1, WHITE_KING=-2, EMPTY=0`
- **AI**: Alpha-beta minimax + транспозиционная таблица (Zobrist), iterative deepening, quiescence search, LMR, killer moves
- **UI**: минималистичный — только доска + стандартное меню Windows
- **Фреймворк**: PyQt6 + NumPy
- **Язык интерфейса**: только русский
- **Сохранения**: JSON
- **Автор**: Andrey Dugin — в «О программе»

---

## Ключевые факты о коде

- Тёмные клетки: `x % 2 != y % 2` → 32 активные клетки
- `y=0` — верх (чёрные), `y=7` — низ (белые); нотация `(0,0)=a8`
- Русские шашки: обязательное взятие, множественные прыжки, дамки на любое расстояние
- AI: `AIEngine.find_move()` → `_search_best_move()` (iterative deepening) → `_alphabeta()` (TT + LMR + killers) → `_quiescence()` → `_evaluate_fast()`
- Оценка: материал + продвижение + центр + связанные пешки + фазовые веса + расстояние дамки + золотые углы
- Детекция повторов и ничейных эндшпилей

## Версионирование

- Единственный source of truth: `draughts/__init__.py` → `__version__`
- При добавлении новой функциональности или существенных изменениях — **инкрементировать версию**:
  - Патч (3.0.x): баг-фиксы, мелкие правки
  - Минор (3.x.0): новая функциональность, улучшения AI, новые настройки
  - Мажор (x.0.0): только по явному запросу пользователя
- Версию обновлять **в том же коммите**, где вносятся изменения
- При обновлении версии — **обязательно обновить CHANGELOG.md** (добавить новую секцию с описанием изменений)
