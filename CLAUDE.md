## Ключевые факты об оригинальном коде

### Доска
- `array[1..8, 1..8] of char`, индексация `field[y, x]`
- Используются только тёмные клетки: `x mod 2 ≠ y mod 2` → 32 активные клетки
- Символы: `'b'`=чёрная шашка, `'B'`=чёрная дамка, `'w'`=белая шашка, `'W'`=белая дамка, `'n'`=пусто
- Начальная позиция (32 символа): `bbbbbbbbbbbbnnnnnnnnwwwwwwwwwwww`
- `y=1` — верх (чёрные), `y=8` — низ (белые)
- Нотация: столбцы `a–h`, строки `8–1` сверху вниз → `(x=1,y=1)=a8`, `(x=8,y=8)=h1`

### Правила
- Русские шашки: шашки ходят только вперёд, дамки — на любое расстояние
- Обязательное взятие, множественные прыжки, взятая фигура убирается после серии
- Превращение в дамку при достижении последнего ряда

### ИИ (компьютер играет чёрными)
Три уровня приоритета, вызываются последовательно:
1. **`SeeBeat()`** — обязательные взятия: Monte Carlo (1000 итераций), выбирает серию с максимальным счётом
2. **`Combination()`** — тактические жертвы: ищет ход, после которого белые вынуждены взять, и проверяет, выгоден ли ответный удар
3. **`Action()`** — обычные ходы: перебор с эвристической оценкой (для дамок тоже Monte Carlo)

Самообучение: сохраняет хорошие/плохие позиции в `goodbase.dat` / `badbase.dat` (текстовые файлы, по 32 символа на строку).

### Формат сохранений (`.sav`)
```
1              ← difficulty (1/2/3)
1              ← speed (1/2/3)
TRUE           ← remind (подсказка взятия)
1              ← language (1=рус, 2=eng)
FALSE          ← soundeffect
0.750          ← pause (коэффициент задержки)
<32 символа>   ← позиция после хода 0 (начало игры)
<32 символа>   ← позиция после хода 1
...
               ← пустая строка
<32 символа>   ← movie (те же позиции для просмотра)
...
```

## Соглашения для переписывания

- Язык: **Python 3.12+**
- Оригинальную логику игры и ИИ сохранить точно; не улучшать алгоритм без явной просьбы
- Для графики ориентироваться на **pygame** (аналог BGI + мышь в одной библиотеке)
- Совместимость форматов `.sav` с оригиналом — желательна (чтобы старые сохранения загружались)
- `IZVRAT.PAS` (10×10) — реализовывать только после завершения 8×8 и по отдельному запросу
