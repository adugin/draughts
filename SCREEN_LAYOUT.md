# Экранная раскладка DRAUGHTS (640x480, VGA)

> Восстановлено по скриншоту DOS-версии и анализу исходного кода (`DRAUGHTS.PAS`, `COMANDER.PAS`).  
> Все координаты — абсолютные пиксели для разрешения **640x480** (`getmaxx=639`, `getmaxy=479`).

---

## 1. Глобальные константы

```
width  = 40     — размер одной клетки доски (в пикселях)
border = 2      — внутренний отступ от границы клетки до заливки
x0     = 8      — число столбцов
y0     = 8      — число строк
stroka0 = 9     — начальная строка вывода нотации (в текстовых единицах)
```

VGA-константы: `getmaxx = 639`, `getmaxy = 479`.

## 2. Общая схема экрана

Экран делится на **три основные области**:

```

│                                                  │                     │
│                 ЗАГОЛОВОК (17..382, 1..20)        │  ПРАВАЯ ПАНЕЛЬ      │
│  "Автор и разработчик программы: Дугин Андрей"   │  (385..639, 1..383) │
│──────────────────────────────────────────────────│                     │
│                                                  │  Шашки-образцы      │
│          РАМКА ДОСКИ                             │  Нотация            │
│    (жёлтая рамка: 20,21 .. 360,361)             │  Кнопки             │
│                                                  │  Таймер             │
│          ИГРОВАЯ ДОСКА                           │  Часы, Дата         │

│                                                                        │

│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 3. Структурные панели (Button-контейнеры)

Фоновые панели создаются вызовом `Button(..., '', true)` — кнопки без текста, выполняющие роль контейнеров:

| Элемент | x1 | y1 | x2 | y2 | Описание |
|---------|----|----|----|----|----------|
| Правая панель | 385 | 1 | 639 | 383 | Контейнер для образцов шашек, нотации, кнопок |
| Нижняя панель | 17 | 385 | 639 | 479 | Контейнер для сообщений и сбитых шашек |

Обе панели — это `lightgray`-кнопки с 3D-рамкой (белая линия сверху/слева, тёмно-серая снизу/справа, чёрный прямоугольник по краю).

## 5. Рамка доски

```pascal
setlinestyle(solidln, 0, 3);     // толщина линии = 3
setcolor(red);                    // RGB переназначен: (63,63,22) ≈ жёлтый
rectangle(width div 2, width div 2 + 1, (x0+1)*width + width div 2, (y0+1)*width + width div 2);
```

Вычисленные координаты:
- **Левый верхний угол**: (20, 21)
- **Правый нижний угол**: (380, 361)
- **Толщина**: 3 пикселя
- **Цвет**: жёлтый (`red` с переназначенной палитрой RGB(63,63,22))

Внутри рамки — зелёная заливка:

```pascal
setfillstyle(solidfill, green);   // RGB переназначен: (25,0,15) ≈ тёмно-зелёный/бордовый

```

- **Заливка**: (23, 25) — (377, 358)
- Эта зелёная область видна в углах доски и за буквами/цифрами нотации

## 6. Игровая доска (шахматное поле)

### Расположение клеток

Клетки рисуются вызовом `Square(x, y, color)`:

```pascal
procedure Square(x, y: integer; color: word);
begin
  rectangle(x*width, y*width, (x+1)*width, (y+1)*width);  // чёрная рамка, толщина 3
  bar(x*width+border, y*width+border, (x+1)*width-border, (y+1)*width-border);  // заливка
end;
```

- Координаты клетки `[x, y]` (1-индексация):
  - **Левый верхний угол**: `(x*40, y*40)`
  - **Правый нижний угол**: `((x+1)*40, (y+1)*40)`
  - **Заливка**: от `(x*40+2, y*40+2)` до `((x+1)*40-2, (y+1)*40-2)` — то есть 36x36 px
- Поле: x от 1 до 8, y от 1 до 8
- **Пиксельная область всех клеток**: X от 40 до 360, Y от 40 до 360 (8x8 клеток по 40px)
- **Светлые клетки**: `yellow` (x mod 2 = y mod 2)
- **Тёмные клетки**: `brown` (x mod 2 ≠ y mod 2) — на них стоят шашки

### Таблица пиксельных координат клеток

| y\x | 1 (a) | 2 (b) | 3 (c) | 4 (d) | 5 (e) | 6 (f) | 7 (g) | 8 (h) |
|-----|-------|-------|-------|-------|-------|-------|-------|-------|
| 1 (8) | 40,40 | 80,40 | 120,40 | 160,40 | 200,40 | 240,40 | 280,40 | 320,40 |
| 2 (7) | 40,80 | 80,80 | 120,80 | 160,80 | 200,80 | 240,80 | 280,80 | 320,80 |
| 3 (6) | 40,120 | 80,120 | 120,120 | 160,120 | 200,120 | 240,120 | 280,120 | 320,120 |
| 4 (5) | 40,160 | 80,160 | 120,160 | 160,160 | 200,160 | 240,160 | 280,160 | 320,160 |
| 5 (4) | 40,200 | 80,200 | 120,200 | 160,200 | 200,200 | 240,200 | 280,200 | 320,200 |
| 6 (3) | 40,240 | 80,240 | 120,240 | 160,240 | 200,240 | 240,240 | 280,240 | 320,240 |
| 7 (2) | 40,280 | 80,280 | 120,280 | 160,280 | 200,280 | 240,280 | 280,280 | 320,280 |
| 8 (1) | 40,320 | 80,320 | 120,320 | 160,320 | 200,320 | 240,320 | 280,320 | 320,320 |

> Каждая клетка занимает 40x40 px. Координаты — левый верхний угол.  
> В скобках — шахматная нотация (цифры строк `8..1` сверху вниз, буквы столбцов `a..h`).

### Центры клеток (для шашек)

Шашка в клетке `[x, y]` рисуется в центре: `(x*40 + 20, y*40 + 20)`.

## 7. Координатные подписи (нотация на доске)

### Буквы столбцов (a–h)

```pascal
outtextxy(x*width + width div 2, 3*width div 4 - 2, abc[x]);     // сверху
outtextxy(x*width + width div 2, (y+1)*width + width div 8, abc[x]); // снизу (y=8)
```

- **Сверху**: Y = 28 (3*40/4 - 2), X = центр столбца (60, 100, 140, ..., 340)
- **Снизу**: Y = 365 (9*40 + 5), X = то же
- **Цвет**: yellow
- **Выравнивание**: `settextjustify(0, 2)` — left, bottom

### Цифры строк (8–1)

```pascal
outtextxy(3*width div 4 - 2, y*width + width div 2, digits[y]);   // слева
outtextxy((x0+1)*width + 6, y*width + width div 2, digits[y]);    // справа
```

- **Слева**: X = 28, Y = центр строки (60, 100, 140, ..., 340)
- **Справа**: X = 366 (9*40 + 6), Y = то же
- **Цвет**: yellow
- **Массив**: `digits = '87654321'` — для y=1 выводится '8', для y=8 выводится '1'

## 8. Шашки (Draughtsman)

### Размер и позиция

```pascal
r := (width - 4 - border) div 2;   // r = (40-4-2)/2 = 17
```

- **Радиус шашки**: 17 пикселей
- **Центр**: `(x*40 + 20, y*40 + 20)` — совпадает с центром клетки

### Обычная шашка

1. Закрашенный сектор (круг) `sector(cx, cy, 0, 360, r, r)` цветом шашки (white/black)
2. Внешний круг `circle(cx, cy, 17)` — серая обводка (darkgray для белых, lightgray для чёрных)
3. Концентрические кольца: `circle(cx, cy, 13)`, `circle(cx, cy, 9)` — те же цвета обводки
4. Кольцо `circle(cx, cy, 5)` рисуется только если `width >= 50` (при width=40 — **не рисуется**)

### Дамка

1. Закрашенный сектор цветом шашки
2. Серая обводка внешнего круга
3. Вместо колец — **корона** (7 линий, образующих зубчатую фигуру):
   - Основание: `(cx-8, cy+7)` — `(cx+8, cy+7)`
   - Левая сторона: `(cx-8, cy+7)` — `(cx-10, cy-3)`
   - Правая сторона: `(cx+8, cy+7)` — `(cx+10, cy-3)`
   - Зубцы: `(-10,-3)`→`(-4,+1)`, `(+10,-3)`→`(+4,+1)`, `(-4,+1)`→`(0,-8)`, `(+4,+1)`→`(0,-8)`
4. Заливка короны: `yellow` (floodfill от центра)
5. Три точки-«камня»: `circle(cx, cy+3, 1)`, `circle(cx-5, cy+4, 1)`, `circle(cx+5, cy+4, 1)` — цвет lightmagenta

## 9. Правая панель — верхняя часть

### 9.1 Шашки-образцы (индикаторы текущего хода)

| Элемент | x1 | y1 | x2 | y2 | Описание |
|---------|----|----|----|----|----------|
| Рамка белой шашки | 395 | 10 | 455 | 55 | InButton, lightgray фон |
| Рамка чёрной шашки | 460 | 10 | 520 | 55 | InButton, lightgray фон |

Шашки рисуются в центрах:
- **Белая**: `Draughtsman(425, 33, false, white)` — x=(395+455)/2=425, y≈33
- **Чёрная**: `Draughtsman(490, 33, false, black)` — x=(460+520)/2=490, y≈33

Эти шашки показаны на скриншоте в правом верхнем углу правой панели.

### 9.2 Панели нотации ходов

| Элемент | x1 | y1 | x2 | y2 | Описание |
|---------|----|----|----|----|----------|
| Левая колонка (белые) | 394 | 60 | 456 | 375 | InButton, чёрный фон |
| Правая колонка (чёрные) | 460 | 60 | 521 | 375 | InButton, чёрный фон |

- **Размеры**: 62x315 px (каждая)
- **Текст нотации**: выводится белым/серым по центру каждой колонки
  - Белые: `outtextxy((394+456) div 2, stroka*textheight(' '), ...)` → X=425
  - Чёрные: `outtextxy((460+521) div 2, stroka*textheight(' '), ...)` → X=490
  - (совпадает с центрами шашек-образцов)
- **Начальная строка**: `stroka0=9` → Y = 9 * textheight(' ')
  - В шрифте 0 размер символа = 8px, `textheight(' ')` = 8 → Y начала = 72
- **Шаг**: `delta=2` строки (16 px) между ходами
- **Предел**: `limit=44` строк → Y max = 44*8 = 352
- При переполнении — **скроллинг**: верхняя часть сдвигается вверх

### 9.3 Кнопки управления

Все кнопки расположены в правой части экрана:

| # | x1 | y1 | x2 | y2 | Текст (рус.) | Текст (англ.) | Высота | Активна по умолчанию |
|---|----|----|----|----|-------------|---------------|--------|---------------------|
| 1 | 530 | 10 | 629 | 40 | Отмена хода | Undo | 30 | Нет (серый текст) |
| 2 | 530 | 40 | 629 | 70 | Открыть | Open | 30 | Да |
| 3 | 530 | 70 | 629 | 100 | Сохранить | Save | 30 | Нет (серый текст) |
| 4 | 530 | 100 | 629 | 130 | Информация | Info | 30 | Да |
| 5 | 530 | 130 | 629 | 160 | Опции | Options | 30 | Да |
| 6 | 530 | 160 | 629 | 190 | Просмотр | Playback | 30 | Да |
| 7 | 530 | 190 | 629 | 220 | Развитие | Development | 30 | Да |
| 8 | 530 | 220 | 629 | 250 | Новая игра | New Game | 30 | Да |

> `x2 = getmaxx - 10 = 629`. Ширина каждой кнопки = 99 px, высота = 30 px.

Неактивные кнопки (`permission=false`): текст рисуется с 3D-эффектом — белая тень +1px, серый основной текст. Активные (`permission=true`): чёрный текст.

```pascal
Button(width-23, 385, getmaxx, getmaxy, '', true)   // (17, 385) — (639, 479)
```

### 11.1 Область сообщений / сбитых шашек

```pascal
InButton(width-13, 395, 520, getmaxy-10, cyan)     // (27, 395) — (520, 469)
```

- **Координаты**: (27, 395) — (520, 469)
- **Размер**: 493 x 74 px
- **Цвет фона**: `cyan` (палитра переназначена: RGB(0,15,0) ≈ тёмно-зелёный)
- Используется для:
  - **Сбитых шашек** (отображаются маленькими шашками)
  - **Сообщений ИИ** (юмористические фразы из MESSAGE.TXT) — всплывающее окно поверх

### 11.2 Сбитые шашки

Сбитые белые шашки (верхний ряд):

```pascal
draughtsman(beat_w*width + width div 2 - 7, (y0+2)*width + 12 + 1, false, white)
```
- Y = 10*40 + 13 = **413** (постоянная)
- X = beat_w * 40 + 13 → для 1-й сбитой: 53, для 2-й: 93, и т.д.
- Шашки идут с шагом 40 px

Сбитые чёрные шашки (нижний ряд):

```pascal
draughtsman(beat_b*width + width div 2 - 7, (y0+3)*width + 11 - 1, false, black)
```
- Y = 11*40 + 10 = **450** (постоянная)
- X = beat_b * 40 + 13 → аналогично

Максимум 12 сбитых шашек каждого цвета → до X=12*40+13=493 (вписывается в область 520 px).

### 11.3 Всплывающее сообщение (Message)

```pascal
procedure Message;
  x1 := 274 - textwidth(msg) div 2 - 10;
  y1 := 443 - 36;     // = 407
  window(x1, y1, textwidth(msg)+20, 46, 'Сообщение', false);

```

- **Центр текста**: X=274, Y=439
- **Ширина окна**: зависит от длины сообщения + 20 px отступ
- **Высота окна**: 46 px
- **Верхний край**: Y=407
- Отображается поверх нижней панели на время, пока ИИ «думает»

## 12. Элементы UI — стиль и принципы

### 12.1 Кнопка (Button)

```
┌─────────────────────────────────┐ ← чёрный прямоугольник (rectangle)
│┌───────────────────────────────┐│ ← белые линии (свет, 1-2px)
││                               ││
││         ТЕКСТ КНОПКИ          ││
││                               ││
│└───────────────────────────────┘│ ← тёмно-серые линии (тень, 1-2px)
└─────────────────────────────────┘
```

- **Фон**: `lightgray`
- **Свет** (верх + лево): `white` линии, 1 или 2 пикселя толщиной
  - Для маленьких кнопок (X, Да, Нет) — 1 px
  - Для остальных — 2 px (двойная линия)
- **Тень** (низ + право): `darkgray`, аналогичная толщина
- **Рамка**: `black` rectangle по периметру
- **Текст**: центрирован (`settextjustify(1,1)`)
  - Активная (`permission=true`): чёрный текст
  - Неактивная (`permission=false`): белая тень (смещение +1,+1), тёмно-серый текст сверху

### 12.2 Нажатие кнопки (ButtonPress)

При нажатии кнопка перерисовывается с **инвертированным** 3D-эффектом:
- Свет меняется местами с тенью (белый → снизу/справа, серый → сверху/слева)
- Текст смещается на 1px вниз: `outtextxy(cx, cy+2, text)` вместо `cy+1`
- Кнопка возвращается в исходное состояние после отпускания мыши

### 12.3 InButton (утопленная панель)

```
┌─────────────────────────────────┐
│█████████████████████████████████│ ← darkgray линии сверху и слева
│█                               │
│█          содержимое           │
│                               █│
│                               █│ ← white линии снизу и справа  
└─────────────────────────────────┘
```

Противоположный 3D-эффект по сравнению с Button — создаёт впечатление «вдавленной» области.

### 12.4 Модальное окно (Window)

```
┌─══════════════════════════════╤──┐  ← Button-рамка (3D)
│  Синий заголовок              │X │  ← bar(blue), кнопка закрытия
├───────────────────────────────┴──┤
│                                  │
│         Содержимое окна          │  ← lightgray фон
│                                  │
└──────────────────────────────────┘
```

- **Анимация открытия**: прямоугольник растёт от центра к полному размеру
- **Заголовок**: синяя полоса высотой `d=16` px, белый текст слева
- **Кнопка [X]**: 14x13 px в правом верхнем углу заголовка
  - Координаты: `(x+width-border-15, y+border+2)` — `(x+width-border-1, y+border+15)`
- **border**: обычно 3px

|--------|---------------|------------|-----------------|-----------------|
| red (4) | Рамка доски | (63, 63, 22) | Жёлто-золотистый | `rectangle` рамки вокруг доски |
| green (2) | Фон вокруг доски | (25, 0, 15) | Тёмный бордово-зелёный | `bar` заливки области вне клеток |
| cyan (3) | Нижняя панель | (0, 15, 0) | Тёмно-зелёный | InButton нижней панели |

> **Важно**: цвета `red`, `green`, `cyan` в DRAUGHTS не соответствуют своим стандартным значениям!  
> При переносе на Python нужно использовать реальные RGB, а не имена констант.

## 14. Шрифт

Весь интерфейс использует **шрифт 0** (Default Font) Borland BGI:
```pascal
settextstyle(0, 0, 1);
```

- **textheight(' ')** = 8
- **textwidth(' ')** = 8

## 15. Курсор мыши

### Ограничения движения

```pascal
setxrange(1, getmaxx - 16);   // 1..623
setyrange(1, getmaxy - 16);   // 1..463
```

Курсор не может выйти за пределы экрана минус 16 пикселей (размер курсора мыши в DOS).

### Выделение клетки

Когда игрок кликает по своей шашке:
```pascal
setcolor(cursor);   // lightmagenta
rectangle(x1*width, y1*width, (x1+1)*width, (y1+1)*width);
```

Клетка обрамляется `lightmagenta` прямоугольником. При множественном взятии промежуточные клетки обрамляются `lightgreen`.

## 18. Обработка кликов — зоны интерактивности

### Доска

```
if (xm >= width) and (xm <= (x0+1)*width) and (ym >= width) and (ym <= (y0+1)*width)
```
→ зона: (40, 40) — (360, 360)

Координаты клетки из пикселей: `x = xm div 40`, `y = ym div 40` (1-индексация автоматическая).

### Кнопки правой панели

Каждая кнопка: `(xm >= 530) and (ym >= Y1) and (xm < 629) and (ym < Y2)`.

### Заголовок (об авторе)

```
(xm >= 17) and (ym >= 1) and (xm <= 382) and (ym <= 20)
```

## 19. Рекомендации для переноса на Python (PyQt)

### Масштабирование

Оригинальное разрешение 640x480 будет мелким на современных экранах. Рекомендуется:
- Ввести коэффициент масштабирования `SCALE` (например, 1.5 или 2.0)
- Все координаты умножать на `SCALE`
- Размер окна: `640*SCALE x 480*SCALE`

### Замена элементов BGI → PyQt

| BGI | PyQt эквивалент |
|-----|-----------------|
| `bar(x1,y1,x2,y2)` | `QPainter.fillRect()` |
| `rectangle(x1,y1,x2,y2)` | `QPainter.drawRect()` |
| `circle(x,y,r)` | `QPainter.drawEllipse()` |
| `sector(x,y,0,360,rx,ry)` | `QPainter.drawEllipse()` + `setBrush()` |
| `outtextxy(x,y,text)` | `QPainter.drawText()` |
| `line(x1,y1,x2,y2)` | `QPainter.drawLine()` |
| `floodfill(x,y,color)` | `QPainterPath` + `fillPath()` |
| `Button(...)` | Можно использовать `QPushButton` с custom style, или рисовать кастомно |
| `InButton(...)` | `QFrame` с `QFrame.Sunken` стилем, или кастомная отрисовка |
| `window(...)` | `QDialog` с кастомным titlebar |
| `getpixel/putpixel` | `QImage.pixel()` / `setPixel()` — для анимаций удаления шашек |
| `setrgbpalette` | Прямое задание `QColor(r, g, b)` вместо индексов палитры |

### Кнопки

Два подхода:
1. **QPushButton + StyleSheet** — проще, но меньше контроля над 3D-эффектом
2. **Кастомная отрисовка** — точное воспроизведение оригинального стиля через `QPainter`

Рекомендация: для максимальной точности использовать кастомную отрисовку, повторяющую логику `Button()` и `ButtonPress()`.

### Шрифт

BGI Default Font — моноширинный, 8x8 px. В PyQt можно использовать:
- `QFont("Courier New", 8 * SCALE)` или
- `QFont("Consolas", 8 * SCALE)`
- Или bitmap-шрифт для точного воспроизведения

### Анимации

Оригинал содержит анимации:
- **Открытие окна**: прямоугольник растёт от центра (задержка ~1.5ms на шаг * pause)
- **Удаление шашки**: попиксельное стирание в цвет клетки (для дамок — random) или плавная смена палитры (для обычных)
- **Эффект молнии**: при заставке (`thunder`)
- **Бегущая строка**: в окне «Информация»

В PyQt анимации реализуются через `QPropertyAnimation` или `QTimer` для покадровой отрисовки.

### Таймер

Вместо `gettime` из DOS использовать `QTimer` с интервалом 1 секунда.
