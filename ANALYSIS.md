# Анализ программы DRAUGHTS

> Программа написана на **Borland Pascal 7.0**, ориентировочно в 1998–2000 годах.  
> Автор: **Дугин Андрей** (упоминается в коде и в окне «Сведения о программе»).  
> Адрес автора, указанный в коде: г. Губкин Белгородской области, ул. Революционная 55-4, тел. (82151) 4-15-15.

---

## 1. Что делает программа

**DRAUGHTS** — это полноценная графическая игра в **русские шашки** для DOS/Windows 3.x с режимом «человек против компьютера».

- Компьютер играет **чёрными** (ходит первым после первого хода игрока, или сразу, если `invertcolor=true`).
- Человек играет **белыми**.
- Управление — мышь (через INT 33h).
- Графика — режим EGA/VGA 640×480, библиотека Borland Graph (BGI).
- Интерфейс полностью на русском языке (есть переключение на английский в настройках, но строки в коде всегда русские — переключение, судя по всему, не реализовано до конца).

## 3. Представление доски

### Тип данных

```pascal
type chessboard = array[1..8, 1..8] of char;
```

Двумерный массив символов. Индексация: `field[y, x]`, где `y` — строка (1 = верх), `x` — столбец (1 = лево).

### Значения символов

| Символ | Значение |
|--------|----------|
| `'b'` | Чёрная шашка |
| `'B'` | Чёрная дамка |
| `'w'` | Белая шашка |
| `'W'` | Белая дамка |
| `'n'` | Пустая клетка |

### Активные клетки

Используются только клетки, где `x mod 2 ≠ y mod 2` (тёмные клетки при нумерации от 1). Это 32 клетки из 64.

### Начальная расстановка

```
bbbbbbbb   ← строки y=1,2,3 (сверху): чёрные шашки
bbbbbbbb
bbbbbbbb
nnnnnnnn   ← строки y=4,5: пусто
nnnnnnnn
wwwwwwww   ← строки y=6,7,8 (снизу): белые шашки
wwwwwwww
wwwwwwww
```

В виде 32-символьной строки (только тёмные клетки, построчно справа налево): `bbbbbbbbbbbbnnnnnnnnwwwwwwwwwwww`

### Нотация ходов

Столбцы: `a–h` (слева направо, `abc[x]`).  
Строки: `8–1` (сверху вниз, `digits[y]` = `'87654321'`).  
Таким образом, `(x=1,y=1) = a8`, `(x=8,y=8) = h1`.  
Формат хода: `e3-f4`.

## 4. Правила игры

Реализованы **русские шашки**:

1. **Шашки ходят** по одной клетке по диагонали вперёд (белые — вверх, чёрные — вниз).
2. **Обязательное взятие**: если возможно бить — обязан бить. При нарушении шашка, которой можно было бить, снимается без хода (`confiscate`). Игрок получает предупреждение.
3. **Взятие** — прыжок через фигуру противника на свободную клетку за ней; взятая фигура остаётся на доске до окончания серии взятий, затем снимается (`away()`).
4. **Множественное взятие**: серия прыжков за один ход (правая кнопка мыши фиксирует промежуточные позиции).
5. **Дамка**: шашка, достигшая последней горизонтали, превращается в дамку (`'w'→'W'`, `'b'→'B'`).
6. **Дамка ходит** на любое количество клеток по диагонали (русские шашки).
7. **Конец игры**: все фигуры одной стороны сбиты, либо у стороны нет ходов — функция `Continue()`.

## 6. Алгоритм ИИ (компьютер играет чёрными)

### Вызов функции `Computer()`

1. Считать случайную фразу из `message.txt` → показать в окошке «Думаю…».
2. Запустить **`SeeBeat()`** — обязательные взятия.
3. Если ходов не было → **`Combination()`** — тактические жертвы.
4. Если ходов нет → **`Action()`** — обычный ход.
5. Обучение: сохранить позицию в `goodbase.dat` / `badbase.dat` при выгодном/невыгодном развитии.

---

### 6.1. `SeeBeat()` — выбор взятия

Если на доске есть обязательные взятия для чёрных:

1. Для каждой чёрной шашки/дамки, у которой есть взятие, запускается **Монте-Карло**: `maxrnd=1000` случайных симуляций.
2. В каждой симуляции: из текущей позиции случайно выбираются последовательные прыжки (пока можно бить) → получается итоговая позиция.
3. Для каждой симуляции вычисляется **оценка**:
   - `+1` за каждую сбитую фигуру
   - `+2` за взятие дамки или за превращение в дамку
   - `+1` если итоговая позиция шашки не находится под боем
   - `+3` / `-3` за нахождение позиции в `goodbase.dat` / `badbase.dat`
4. Запоминается последовательность ходов с максимальной оценкой → именно её компьютер и выполняет.

---

### 6.2. `Combination()` — тактические жертвы

Для обычных шашек (не дамок) среди числом > 1:

1. Проверяем ход вперёд (в одну из двух диагоналей, только вперёд), перемещая шашку на одну клетку.
2. После «жертвенного» хода вызывается `OneBeat()`: проверяется, есть ли **ровно одно** взятие у белых (т.е. белые вынуждены взять именно эту шашку).
3. Если да — симулируется: белые берут (`VirtualWhiteBeat`), затем чёрные берут (`VirtualBlackBeat`), глубина = `depth0=1`.
4. Оценка `score = -delta_white + delta_black`. Если `score > 2` — ход выгоден, выполняется.

---

### 6.3. `Action()` — обычный ход

Для каждой чёрной шашки/дамки перебираются все возможные ходы:

**Для обычной шашки** (вперёд, одна диагональ, 2 направления):
- `+0.5` если новая позиция «рядом» с краем (функция `neighbour`)
- `+1` если был ход по краю и после него перестал
- `+2` за достижение последнего ряда (стать дамкой)
- `+1` если ушли с опасной позиции
- `+1.5` / `-2` если новая позиция безопасна/опасна (`dangerousposition`)
- `+3` / `-3` за совпадение с `goodbase` / `badbase`
- Одношаговое заглядывание вперёд: `VirtualWhiteBeat` / `VirtualBlackBeat`

**Для дамки** (`maxrnd=1000` случайных направлений и расстояний):
- `+1` если дамка ставится под угрозу белой дамки
- `+1` если уходим с опасной позиции
- `+2` за уход с опасности при безопасном месте назначения
- `-3` если новая позиция под боем
- `+3` / `-3` база позиций
- Заглядывание вперёд

Выбирается ход с максимальной суммарной оценкой (при равенстве — случайно).

### 6.4. Вспомогательные функции ИИ

| Функция | Описание |
|---------|----------|
| `DangerousPosition(x,y,mas,color)` | Проверяет, стоит ли фигура под боем |
| `DangerousBeat(x1,y1,x2,y2,bx,by,mas,color)` | Проверяет, попадёт ли фигура под бой после прыжка |
| `VirtualBlackBeat(model)` | Симулирует лучшее взятие чёрных на копии доски |
| `VirtualWhiteBeat(model)` | То же для белых |
| `OneBeat(mas)` | Возвращает `true` если у чёрных ровно одно возможное взятие |
| `compare(mas1,mas2)` | Сравнивает ценность двух позиций |
| `appreciate(mas1,mas2,color)` | Оценивает выгодность изменения позиции |
| `freeway(x1,y1,x2,y2,mas)` | Проверяет, свободен ли диагональный путь между двумя клетками |
| `exist(x1,y1,x2,y2,bx,by,color,mas)` | Проверяет, есть ли на пути ровно одна фигура (для прыжка) |
| `canbeat(x1,y1,d,r,mas)` | Пытается выполнить взятие в направлении `d` на `r` шагов |

### 6.5. Обучение (самообучающаяся база)

После каждого хода компьютера и после каждого хода игрока:

- **Победа чёрных** (`blackwin=true`) + позиция изменилась выгодно → последние 3 позиции записываются в `goodbase.dat`.
- **Проигрыш чёрных** (`whitelose=true`) + позиция ухудшилась → записывается в `badbase.dat` (через `invertstr` — в инвертированном виде, для симметрии).

Базы — текстовые файлы, каждая строка — 32-символьная строка состояния доски (только тёмные клетки). При поиске: `search(filename, getstring(model))`.

## 7. Ввод игрока (белые)

### Выбор шашки

1. Левый клик по белой шашке/дамке на доске.
2. Вокруг неё рисуется рамка цвета `lightmagenta` (курсор выбора).
3. Если есть обязательное взятие и `remind=true` → кратное мигание рамки с звуком.

### Выполнение хода

- **Левый клик** по целевой клетке — обычный ход (шашка двигается в пустую клетку).
- **Правый клик** по промежуточным клеткам — фиксация маршрута серийного взятия; **левый клик** по конечной клетке — подтверждение.
- Рамка промежуточных клеток рисуется цветом `lightgreen`.

### Валидация хода

- Обычная шашка: ходит только вперёд, на 1 клетку по диагонали.
- При взятии: на 2 клетки, между ними должна стоять фигура противника.
- Дамка: ходит на любое расстояние, при взятии — через одну фигуру на любое расстояние.
- Обязательное взятие: функция `greenlight(x,y)` — если есть хоть одно взятие, запрещает обычные ходы.

## 11. Инициализация и главный цикл

### Запуск

```
1. Установить параметры по умолчанию
2. init(n1,n2)        ← инициализация мыши
3. initgraph(a,c,'')  ← открыть графический режим EGA/VGA
4. disappear()         ← заготовить шумовой экран («снег») для анимации
5. screen()            ← заставка с именем автора
6. board(false)        ← нарисовать начальную доску
7. Читать draughts.his ← восстановить имя последнего файла и pause
```

### Главный цикл (`repeat ... until false`)

```
1. Если у белых нет ходов → OnceAgain → новая игра или выход
2. Если invertcolor=true и ещё не начали → Computer() (ходит первым)
3. Ждать клик мышью (getmouseinfo)
   ├─ Клик по доске + выбрана белая фигура + greenlight → выделить шашку
   ├─ Клик по кнопкам панели → выполнить соответствующее действие
   └─ Выбор целевой клетки → Player() → ход белых
4. Player() выполнен → Computer() → ход компьютера
5. AutoSave() → сохранить позицию в autosave.sav
6. Проверить конец игры:
   ├─ Нет фигур одной из сторон → OnceAgain
   ├─ Компьютер не нашёл хода → OnceAgain
   └─ Продолжать
```

## 12. Файл IZVRAT.PAS

Практически идентичен `DRAUGHTS.PAS`, за следующими отличиями:

- `const x0=10; y0=10;` — доска 10×10 (50 активных клеток)
- `type chessboard=array[1..10,1..10] of char;`
- Функция `digits` объявлена с длиной 10, но значение всё равно `'87654321'` (8 символов) — вероятная ошибка

**Проблема совместимости**: юнит `COMANDER.PAS` определяет `chessboard` как `array[1..8,1..8]`. При передаче 10×10-массива в функции `comander` возникает несовместимость типов — вероятно, компилятор выдаёт ошибку или компилируется с предупреждениями. Файл, судя по всему, является незавершённым экспериментом.
